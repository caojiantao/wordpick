## Why

词汇学习类产品普遍功能臃肿、打扰多；**拾词**以「轻量对照词表」为起点，用现成的分级词库 JSON 在手机浏览器里快速可查。需要先冻结首版页面与数据边界，便于用 Vue 3 + Vite 一次性搭出可部署的 H5。

## What Changes

- 定义仅三屏的产品路径：首页（七类入口）→ 类别词列表 → 词条详情。
- 定义词条展示字段：单词、美式/英式音标（若有）、按词性分条的中文释义。
- 数据来源为仓库 `data/` 下七份静态 JSON（初中、高中、CET4、CET6、考研、托福、SAT）；首版按需加载，不引入账号与后端。
- 部署假设为自建环境（例如树莓派）提供静态产物；不要求首版接入公有云。

## Capabilities

### New Capabilities

- `category-home`: 首页展示七类词库入口并可进入对应列表。
- `category-word-list`: 展示当前类别下的单词列表（每行仅单词）；可返回首页并进入词条详情。
- `word-detail`: 展示单个词条全文：单词、`phonetics.us` / `phonetics.uk`、`senses`，并采用基于浏览器历史的返回语义。

### Modified Capabilities

- 无（主规格库为空）。

## Impact

- 新增 Vue 3 + Vite 前端工程与三条路由对应的页面组件；引入 **Vant** 作为移动端 UI 依赖（详见 `design.md`）。
- 运行时从 `data/*.json` 加载词表（可按类别拆分请求）；与未来在同机或其它环境提供 HTTP API 时，仅需替换数据获取层。
- 树莓派或其它机器上仅需 Web 服务器托管 `npm run build` 产物即可；后续动态数据与数据库可在同一宿主上扩展，不影响本变更对首版的约定。
