## Context

**命名：** 产品品牌名为 **「拾词」**（用于界面标题等对外展示）；代码仓库、npm 包名等工程标识为 **`wordpick`**（小写，无空格）。

静态数据拆成两层（类比「底表 + 类别关联表」）：  
- **`data/lexicon.json`**：词形字符串 → 全量词条对象（`word`、`phonetics`、`senses`）；同一词形全局去重，重复时按类别优先级首次出现为准。  
- **`data/lists/<id>.json`**：仅该类下的词形数组（顺序与来源一致），体积小、按需加载。

由 `scripts/build_lexicon.py` 从「按类别全量数组」源文件生成（开发流程见仓库说明）。首版仍离线、无账号。

## Goals / Non-Goals

**目标：**

- 使用 Vue 3 + Vite 构建 SPA，路由覆盖三屏闭环；移动端 UI 使用 **Vant**（与 Vue 3 配套，见下「决策 2」）。
- 类别与静态文件固定映射（见下表），按需加载以避免一次拉齐大文件。
- 自建部署（例如树莓派）：构建产物由 Nginx/Caddy 等静态托管即可。

**非目标（首版）：**

- 用户账号、学习进度、收藏、复习调度、站内搜索。
- 首版不接云端托管；不在本变更内实现后端与数据库（仅预留数据访问抽象以便日后替换为 API）。

## Decisions

### 1. 前端栈：Vue 3 + Vite

单页应用 + 客户端路由；构建输出 `dist/` 交由任意静态服务器提供。

### 2. 移动 UI 组件：Vant

- 采用 **Vant 4**（`vant`）作为 H5 端组件库，负责导航栏、列表（如 `Cell`）、布局等，与手机浏览器触控习惯一致。  
- 通过**按需引入**（如 `unplugin-vue-components` 等）控制包体，首版不整包全量注册。  
- 不引入 Element Plus 等偏桌面组件库于本变更；若日后有 PC 管理端，可另开项目或另起变更。

### 3. 静态词表与加载策略

- 应用首次需要全文时在客户端加载 **`data/lexicon.json`**（一次缓存）；进入某类列表时只加载 **`data/lists/<id>.json`**（词形数组）。词条详情用词形 **关联查询** lexicon。  
- 七类与 `lists` 文件名映射：

| 类别（展示文案可再润色） | 列表文件 |
|--------------------------|----------|
| 初中英语 | `lists/junior.json` |
| 高中英语 | `lists/senior.json` |
| CET4 | `lists/cet4.json` |
| CET6 | `lists/cet6.json` |
| 考研 | `lists/kaoyan.json` |
| 托福 | `lists/toefl.json` |
| SAT | `lists/sat.json` |

后续若改为 REST/数据库，可映射为「词条表 + 类别-词形关联表」；前端契约仍为「列表拿键、详情拿全量」。

### 4. 部署：树莓派（或同类自建主机）静态托管

- 构建：`npm ci && npm run build`，将 `dist/` 置于 Web 服务器根目录（或子路径，由 `base` 配置对齐）。
- 推荐 HTTPS（如 Let’s Encrypt）；仅内网访问时可简化。
- 动态数据与数据库若后续引入，可在**同一台主机**上增加 API 与数据库服务，前端通过同域反代或独立子域访问；本变更不要求首版实现。

### 5. 页面与信息架构

仅三屏：首页（七入口）→ 类别词列表（每行仅 `word`）→ 词条详情（单词 + 两条音标字段 + 全部 `senses`）。无第四屏。

详情页返回采用浏览器上一页语义：若存在应用内上一跳则回退；若为直接访问详情链接，则不强制改写为首页或任意分类列表。

## Risks / Trade-offs

- 词表文件较大，弱网下首次进入某类可能较慢 → 依赖按需加载与浏览器缓存；首版可接受。
- `phonetics.us` 或 `phonetics.uk` 可能为空 → UI 隐藏空字段即可。
